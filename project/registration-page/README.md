# Петербургский кодекс

Интерактивный мини-курс по достопримечательностям Санкт-Петербурга: лендинг,
регистрация/вход, дашборд с главами, путь по эпизодам и квизы.

Проект собран на **чистом HTML + CSS + ES-modules** — без сборщиков и
фреймворков. Хранилище — `localStorage`.

---

## 📂 Структура

```
registration-page/
├── index.html              # Landing page
├── register.html           # Регистрация
├── login.html              # Вход
├── dashboard.html          # Список глав (требует авторизацию)
├── chapter-path.html       # Путь по эпизодам главы (требует авторизацию)
├── episode.html            # Отдельный эпизод (требует авторизацию)
│
├── css/                    # Стили (по одной таблице на компонент/страницу)
├── js/
│   ├── *-main.js           # Точки входа страниц
│   ├── components/         # UI-компоненты (FormTitle, InputField, …)
│   ├── utils/              # auth, theme, validation, animations, progress
│   └── config/             # JSON-конфиги контента (главы, истории, поля форм)
├── assets/icons/           # SVG-иконки
├── NODEBOX/                # Фоновые фотографии достопримечательностей
└── data/users.json         # Не используется в рантайме (см. localStorage)
```

## 🚀 Запуск

Нужен любой статический HTTP-сервер (модули требуют HTTPS/HTTP, не `file://`):

```bash
cd project/registration-page
python3 -m http.server 8000
# открыть http://localhost:8000
```

или

```bash
npx serve project/registration-page
```

## 🔐 Безопасность

> ⚠️ Это демо-проект **без бэкенда**. Все данные пользователя лежат в
> `localStorage` браузера и **уязвимы к XSS**. Не используйте этот код «как есть»
> в продакшене.

Что сделано для разумного минимума:

| Меры | Реализация |
|---|---|
| Хеширование паролей | `PBKDF2-SHA256`, 100 000 итераций, уникальная соль на пользователя (Web Crypto API). |
| Защита от timing-атак | Постоянное по времени сравнение хешей. |
| Auth-guard | Inline-скрипт в `<head>` защищённых страниц + `requireAuth()` в `auth.js` — рендер блокируется до проверки. |
| Миграция legacy-паролей | Старые plain-text пароли автоматически пере-хешируются при первом успешном входе. |
| XSS-экранирование | Все динамические данные (имена глав, имя пользователя) проходят через `escapeHtml()`. |
| Никаких хешей в сессии | `spb_current_user` содержит только `{ id, username, createdAt }`. |

Ключи в `localStorage`:
- `spb_users` — массив зарегистрированных пользователей с хешами.
- `spb_current_user` — текущая сессия (без секретов).
- `spb_theme` — выбранная тема (light/dark).

## 🧱 Архитектура

### Компоненты (`js/components/*.js`)
Каждый компонент — объект с методом `render(props)`, возвращающим HTML-строку.
Состояние компоненты не хранят; точки входа `*-main.js` оркеструют рендер и
подключают события.

### Утилиты (`js/utils/*.js`)
- **`auth.js`** — регистрация, вход, выход, `requireAuth()`, хеширование PBKDF2.
- **`theme.js`** — light/dark + системная тема.
- **`validation.js`** — правила и `getPasswordStrength()` (слабый / средний / сильный).
- **`animations.js`** — тосты, fade in/out, debounce/throttle, escapeHtml.
- **`progress.js`** — прогресс пользователя по главам/эпизодам.
- **`router.js`**, **`story-engine.js`** — навигация по эпизодам и движок историй.

### Конфигурация контента
Главы и истории описаны в `js/config/*.json` — добавить новую главу можно без
правки JS-кода.

## ✨ UX-фишки

- Glassmorphism, кастомные шрифты (Inter, Playfair Display, Montserrat).
- Светлая/тёмная тема с учётом `prefers-color-scheme`.
- Toast-уведомления, скелетоны, плавные анимации.
- Индикатор силы пароля и предупреждение о включённом **Caps Lock** на формах
  регистрации/входа.
- Toggle видимости пароля (👁) — и на регистрации, и на входе.
- Доступность: `aria-*` атрибуты, skip-links, keyboard navigation для карточек.
- Возврат на запрошенную страницу после логина (`sessionStorage.spb_redirect_after_login`).

## 🧪 Ручная проверка

1. **Регистрация**: открыть `register.html`, ввести имя ≥ 3 симв., пароль ≥ 8 симв.
   - Индикатор силы пароля должен меняться в реальном времени.
   - При нажатом Caps Lock должно появляться предупреждение.
2. **Вход**: на `login.html` войти с теми же данными.
   - Проверить toggle видимости пароля.
3. **Защита роутов**: разлогиниться (кнопка в dashboard) → попытаться открыть
   `dashboard.html` напрямую → должен быть мгновенный редирект на `login.html`
   **без вспышки** dashboard.
4. **XSS**: зарегистрировать пользователя с именем `<img src=x onerror=alert(1)>`
   — никаких алертов быть не должно (имя должно отрендериться как текст).
5. **Миграция legacy паролей**: положить в localStorage вручную старую запись
   `{ username: "old", password: "12345678" }` — после входа эта запись должна
   стать `{ username: "old", passwordHash: "pbkdf2$..." }`.
